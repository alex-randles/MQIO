Mapping Quality Improvement Ontology (MQIO): Description back to ToC

Ontology Competency questions have been defined & answered through SPARQL queries: Competency Questions
  • The queries have been executed on a sample graph: Sample Graph
  • The graph was generated by detecting quality issues in the mappings used in the GeoHive Project which uplifts Ireland's National Geospatial Data Hub.
  • A description of the use case has been published: (Paper)
  • The graph has been assessed using RDFUnit (Result).

The SPARQL query shown below can be used to identify detected violations which have not be refined: 
	PREFIX mqio: <https://www.w3id.org/MQIO#>
SELECT ?violation ?refinement
WHERE {
  ?mappingAssessment a mqio:MappingAssessment ;
                     mqio:hasValidationReport ?validationReport . 
  ?validationReport mqio:hasViolation ?violation . 
  FILTER NOT EXISTS { ?violation mqio:wasRefinedBy ?refinement . } 
}

@prefix sh: <://www.w3.org/ns/shacl< .
@prefix xsd: <://www.w3.org/2001/XMLSchema< .
@prefix ex: <://example.org/< .
@prefix mqio: <s://w3id.org/MQIO< .
@prefix dqv: <://www.w3.org/ns/dqv< .

ex:validateQualityViolations
    a sh:NodeShape ;
    sh:targetClass mqio:MappingViolation ;
    sh:property [
		sh:path mqio:wasRefinedBy  ;
		sh:minCount 1;
		sh:message "No refinements associated 
		            with this quality violation." ;
    ] .
